**Это мини-оболочка командной строки, реализующая основные shell-команды для работы с файловой системой: навигация, просмотр, копирование, перемещение, удаление, архивирование и чтение файлов.**
**В проекте использованы стандартные встроенные библиотеки (os, pathlib, shlex, shutil, zipfile, tarfile), а так же модуль pytest для тестирования.**

### Поддерживаемые команды:

- `ls` `[-l]`: показывает список файлов и папок в текущей директории. Флаг -l выводит подробную информацию (права, размер, дата, название).
- `cd` `<путь>`: меняет текущую рабочую директорию.
- `cp` `[-r]` `<источник> <назначение>`: копирует файл или каталог. С флагом -r рекурсивно копирует папки.
- `mv` `<источник> <назначение>`: перемещает или переименовывает файл/папку.
- `rm` `[-r]` `<путь>`: удаляет файл или каталог. Флаг -r позволяет рекурсивно удалять каталоги.
- `cat` `<файл>`: показывает содержимое файла.
- `--help` `<команда>`: краткая справка по командам.
- `zip` `<каталог>` `<название архива>.zip`: создает архив формата ZIP из каталога.
- `unzip` `<архив>.zip`: распаковывает архив ZIP в текущую рабочую директорию.
- `tar` `<каталог>` `<архив>.tar.gz`: создает архив формата TAR.GZ из каталога.
- `untar` `<архив>.tar.gz`: распаковывает архив TAR.GZ в текущую рабочую директорию.
- `grep` `[-r]` `[-i]` `<паттерн>` `<каталог/файл>`: ищет строки по паттерну в файлах и каталогах. Паттерн передается в виде регулярного выражения. Флаг -r для поиска в подкаталогах, -i для поиска без учета регистра.
- `exit`: завершает работу.

### Особенности

- Команды `ls`, `cat` принимают на вход несколько аргументов и делают поочередный вывод с названием каждого файла.
- Команды `cp`, `mv`, `rm` принимают на вход несколько аргументов, но последний обязательно должен быть директорией.
- Команды `zip`, `tar` принимают на вход несколько каталогов, которые будут запакованы в один архив.
- Команды `unzip`, `untar` принимают на вход несколько архивов и распаковывают их в текущую директорию.
- Команда `cd` поддерживает аргумент `..` для перехода на уровень выше, и `~` либо пустой символ (`cd `) для перехода в домашнюю директорию.
- Весь журнал операций сохраняется в файл shell.log с помощью модуля logging.

### Требования
- Python 3.10 или выше
- Установленные пакеты: pytest, pytest-cov, pyfakefs, pytest-mock (для тестирования)

### Запуск программы

```bash
python main.py
```

### Запуск тестов
```bash
uv run pytest --cov-report=term-missing --cov-report=html
```
Данный запуск тестов покажет не только количество пройденных тестов, но и процент покрытия кода этими тестами.

### Ограничения:
- Если для `cp` (или `mv`) передается несколько аргументов, то последний обязательно должен быть директорией, в которую будут копироваться (переноситься) предыдущие файлы и каталоги.
- Запрещено удалять корневую или домашнюю директорию командой `rm [-r]`.
- `zip` и `tar` архивируют только каталоги.

### Обработка ошибок:
- `FileNotFoundError` - файл или директория не найдены
- `NotADirectoryError` - не является директорией
- `PermissionError` - у пользователя недостаточно прав для совершения операций над данным файлом/каталогом
- `IndexError` - передано слишком мало или же слишком много аргументов
- `IsADirectoryError` - является директорией, а не файлом
- `BadZipfile` - переданный аргумент не является архивом формата ZIP
- `BadGzipFile` - переданный аргумент не является архивом формата TAR.GZ
